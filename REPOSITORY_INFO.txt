PROJECT OVERVIEW
================================================================================

An intelligent, AI-powered job application automation system that:
1. Automatically discovers jobs from 4 major platforms
2. Uses OpenAI AI to match your resume with job descriptions
3. Auto-applies to jobs matching your criteria
4. Sends daily Telegram notifications with job statistics
5. Tracks all applications and interviews

Perfect for: Job seekers who want to apply to hundreds of jobs automatically

DATABASE SCHEMA (7 TABLES)
================================================================================

1. users
   - id (UUID, primary key)
   - email (unique)
   - password_hash
   - full_name
   - role
   - preferred_locations (array)
   - employment_type
   - min_match_score
   - created_at, updated_at

2. resumes
   - id (UUID, primary key)
   - user_id (foreign key)
   - file_name
   - file_path
   - text_content
   - skills (array)
   - experience (text)
   - education (text)
   - keywords (array)

3. jobs
   - id (UUID, primary key)
   - title
   - company_name
   - location
   - salary
   - experience_level
   - required_skills (array)
   - job_description
   - apply_url
   - source (linkedin, naukri, indeed, careerPages)
   - external_id (for deduplication)
   - posted_date
   - status

4. applications
   - id (UUID, primary key)
   - user_id (foreign key)
   - job_id (foreign key)
   - status (pending, applied, interview, rejected, offer)
   - applied_at
   - interview_scheduled
   - interview_date
   - rejected
   - offer_received
   - notes

5. match_scores
   - id (UUID, primary key)
   - user_id (foreign key)
   - job_id (foreign key)
   - match_score (0-100)
   - missing_skills (array)
   - matching_skills (array)
   - strengths (text)
   - recommendation (text)
   - analyzed_at

6. automation_settings
   - id (UUID, primary key)
   - user_id (foreign key)
   - auto_apply_enabled (boolean)
   - auto_apply_threshold (integer, 0-100)
   - approval_mode_enabled (boolean)
   - telegram_enabled (boolean)
   - email_notifications (boolean)
   - job_discovery_enabled (boolean)

7. notifications
   - id (UUID, primary key)
   - user_id (foreign key)
   - notification_type (job_found, job_matched, application_applied, interview_scheduled, offer_received)
   - title
   - message
   - related_job_id (nullable)
   - read (boolean)
   - created_at

All tables have indexed on frequently queried fields for performance optimization.

REST API ENDPOINTS (11 TOTAL)
================================================================================

AUTHENTICATION (4 endpoints):
  POST   /api/auth/register           - Create new account
  POST   /api/auth/login              - Login & get JWT token
  GET    /api/auth/profile            - Get user profile
  PUT    /api/auth/profile            - Update user profile

JOBS (3 endpoints):
  GET    /api/jobs                    - List all discovered jobs
  GET    /api/jobs/:jobId             - Get job details
  POST   /api/jobs/search             - Search jobs by keywords

APPLICATIONS (3 endpoints):
  POST   /api/applications/apply      - Apply to a job
  GET    /api/applications            - Get user's applications
  PUT    /api/applications/:id        - Update application status

DASHBOARD (1 endpoint):
  GET    /api/dashboard               - Get statistics & recent jobs

All endpoints require JWT authentication (except login/register).
Rate limiting: 100 requests/15 min (general), 5 requests/15 min (auth).

================================================================================
SCHEDULED JOBS (CRON)
================================================================================

1. Job Discovery Job
   Schedule: Every hour (0 * * * *)
   Action: Scrapes LinkedIn, Naukri, Indeed, CareerPages for new jobs
   Stores: New jobs in database
   Deduplication: Prevents duplicate job entries

2. AI Matching Job
   Schedule: Every hour + 15 minutes (15 * * * *)
   Action: Analyzes unmatched jobs using OpenAI
   Scoring: 0-100% based on resume-job match
   Output: Stores match scores in database

3. Daily Summary Job
   Schedule: Daily at 9 PM (0 21 * * *)
   Action: Sends Telegram notification
   Content: Jobs found, Applications submitted, Interviews, Offers
   Delivery: Via Telegram Bot

QUICK START GUIDE
================================================================================

LOCAL DEVELOPMENT:

1. Clone the repository:
   git clone https://github.com/Mekarthiakhi/autoJobApply.git
   cd ai-job-automation

2. Install dependencies:
   cd backend && npm install
   cd ../frontend && npm install

3. Setup PostgreSQL database locally:
   psql -U postgres -d postgres
   CREATE DATABASE job_automation;

4. Configure environment variables:
   Create backend/.env with your OpenAI API key (already included ✅)
   Create frontend/.env with VITE_API_URL

5. Start the application:
   
   Terminal 1 - Backend:
   cd backend
   npm run dev
   (Runs on http://localhost:5000)
   
   Terminal 2 - Frontend:
   cd frontend
   npm run dev
   (Runs on http://localhost:3000)

6. Access the application:
   Open browser: http://localhost:3000
   Login or create new account
   Upload your resume
   Configure automation settings
   Monitor jobs being discovered and applied

DOCKER DEPLOYMENT:

1. Build and run with Docker Compose:
   docker-compose up -d

2. Access the application:
   Frontend: http://localhost:3000
   Backend API: http://localhost:5000

TROUBLESHOOTING
================================================================================

Q: Backend won't start - "ECONNREFUSED" error
A: PostgreSQL is not running. Start PostgreSQL service or use Docker.

Q: OpenAI API errors
A: Check OPENAI_API_KEY is correct in backend/.env
   Ensure API key has sufficient credits

Q: Jobs not being discovered
A: Check cron jobs are running (Node Cron)
   Verify database connection
   Check job scraper error logs

Q: Telegram notifications not working
A: Set TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID
   Verify bot is active and has permissions
   Check Telegram network connection

Q: High database query times
A: Database indexes are created automatically
   Run ANALYZE on PostgreSQL if slow
   Check database connection pooling

Q: Frontend won't connect to backend
A: Verify VITE_API_URL matches backend running port
   Check CORS settings in backend
   Verify frontend is on http://localhost:3000

Q: Auto-apply not working
A: Check match score threshold (default 80%)
   Verify auto_apply_enabled in automation_settings
   Review scraper implementation for target platform
